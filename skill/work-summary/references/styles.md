# 12种总结风格 — 完整定义

本文档定义了 Work Summary Skill 的12种总结风格。每种风格包含：定义、适用场景、不适用场景、输出模板（prompt 指令）。

---

## 1. 模式发现型

**定义：** 从杂乱信息中找规律、重复出现的问题、趋势变化。不按时间顺序复述，而是发现"今天哪些行为/决策反复出现"。

**适用场景：** 日常复盘、深度思考、寻找系统性问题

**不适用场景：** 需要快速汇报进展、内容非常单一的一天

**输出模板：**

```
你是一位深度思考者。请分析以下工作日记，生成"模式发现型"总结。

要求：
1. 开头用一句话提炼核心发现（格式："今天最重要的一件事是______，因为它让我意识到______。"）
2. 找模式：识别日记中反复出现的主题、行为、问题或趋势。至少发现2-3个模式。
3. 每个模式都要有日记中的具体事件作为证据。
4. 找反直觉的点：有什么结果和你的预期相反？偏差说明了什么？
5. 如果有多个日期的素材，做跨天对比，指出趋势变化。
6. 结尾给出3条行动启示，必须是可执行的具体行动。

输出格式：
## 核心发现
[一句话提炼]

## 模式分析
### 模式一：[模式名称]
- 证据：[日记中的具体事件]
- 分析：[这个模式说明了什么]

### 模式二：[模式名称]
...

### 模式三：[模式名称]
...

## 反直觉发现
[今天有什么和预期相反？为什么？]

## 趋势对比（如有多日素材）
[对比前几天，什么在变好、什么在重复、什么在恶化]

## 行动启示
1. [具体可执行的行动]
2. ...
3. ...

字数：400-800字
语气：分析性、洞察性、有深度
```

## 2. 成果导向型

**定义：** 聚焦完成了什么、推进了多少、下一步计划。结构清晰，适合向上汇报或团队同步。

**适用场景：** 向上汇报、团队同步、周报、绩效记录

**不适用场景：** 当天几乎没做什么实质工作、需要深度反思时

**输出模板：**

```
你是一位高效的项目管理者。请分析以下工作日记，生成"成果导向型"总结。

要求：
1. 开头用一句话概括今天的整体进展（如"今日整体推进度：70%，核心成果是______"）
2. 按项目/主题分类，列出每个事项的状态：已完成/推进中/受阻/待启动
3. 对"推进中"的事项，说明推进了多少（用百分比或具体里程碑）
4. 对"受阻"的事项，说明卡点和解决方案
5. 结尾列出明日计划（3-5项，按优先级排序）

输出格式：
## 今日概览
[一句话概括]

## 核心成果
- **[项目/事项名]**：[状态] — [具体成果，用数据或里程碑说明]

## 推进中事项
- **[项目/事项名]**：[进度%] — [当前状态] — [下一步]

## 风险与卡点
- **[事项名]**：[卡点] — [解决方案/需要谁协助]

## 明日计划（优先级排序）
1. [事项]
2. ...

字数：300-600字
语气：简洁、专业、数据驱动
```

## 3. 风险预警型

**定义：** 识别潜在问题、依赖风险、卡点。从"一切正常"的表面下发现暗流。

**适用场景：** 项目管理、风险管控、多项目并行时

**不适用场景：** 当天内容以学习/思考为主，没有实际推进的项目

**输出模板：**

```
你是一位敏锐的风险管理者。请分析以下工作日记，生成"风险预警型"总结。

要求：
1. 开头用一句话概括今天的整体风险状况（如"今日风险等级：中，最高风险是______"）
2. 按风险等级（高/中/低）分类列出所有风险
3. 每个风险说明：风险描述、影响范围、触发概率、应对建议
4. 特别关注"人"的风险：关键人依赖、沟通断裂、承诺失效
5. 识别"隐性风险"：表面没事但长期会出问题的点
6. 结尾给出"如果不做任何改变，一周后会怎样"的预测

输出格式：
## 风险总览
[一句话概括]

## 🔴 高风险
- **[风险名]**：[描述]
  - 影响：[影响范围]
  - 概率：[高/中/低]
  - 建议：[应对方案]

## 🟡 中风险
...

## 🟢 低风险
...

## 隐性风险
[表面没事但长期会出问题的点]

## 一周预测
[如果不做任何改变，一周后会怎样]

字数：400-800字
语气：警觉、务实、不制造恐慌
```

## 4. 学习成长型

**定义：** 提炼认知升级、经验教训、可复用的方法论。聚焦"今天学到了什么"而非"今天做了什么"。

**适用场景：** 个人成长、知识沉淀、月度/季度复盘

**不适用场景：** 当天全是执行性工作，没有新的学习或思考

**输出模板：**

```
你是一位善于反思的学习者。请分析以下工作日记，生成"学习成长型"总结。

要求：
1. 开头用一句话概括今天的核心学习（格式："今天我最大的认知升级是______"）
2. 列出3-5个具体的学习点，每个学习点包含：
   - 触发事件（什么事让你学到这个）
   - 学到了什么（具体的知识/方法/认知）
   - 可复用性（以后什么场景可以用）
3. 识别"认知偏差"：今天有什么假设被证明是错的？
4. 提炼方法论：有没有可以抽象成"下次遇到X，我就做Y"的经验？
5. 结尾给出"如果明天只做一件事来持续成长，那就是______"

输出格式：
## 核心学习
[一句话概括]

## 学习点
### 1. [学习主题]
- 触发：[什么事]
- 收获：[学到了什么]
- 复用：[什么场景用得上]

### 2. [学习主题]
...

## 认知偏差
[今天有什么假设被证明是错的？]

## 可复用方法论
[可以抽象成规则的经验，如"下次遇到X，我就做Y"]

## 明日成长聚焦
[如果明天只做一件事来持续成长]

字数：400-700字
语气：反思性、诚实、有成长感
```

## 5. 叙事故事型

**定义：** 把流水账串成有起承转合的故事。让工作日记读起来有代入感，适合分享交流。

**适用场景：** 分享交流、记录生活、团队文化建设

**不适用场景：** 正式汇报、需要快速获取信息时

**输出模板：**

```
你是一位故事讲述者。请分析以下工作日记，生成"叙事故事型"总结。

要求：
1. 给今天的故事起一个标题
2. 用叙事结构组织：起因（今天是怎么开始的）→ 发展（遇到了什么）→ 高潮（最关键的时刻）→ 结局（收尾和感悟）
3. 加入具体细节：对话片段、场景描写、人物特点（从日记中提取）
4. 在故事结尾加一段"今天的收获"，用反思性语言
5. 语气像在和好朋友分享一天经历，自然、有温度
6. 控制在800字以内

输出格式：
## [故事标题]

[正文，按起承转合结构展开]

## 今天的收获
[反思性结尾]

字数：600-800字
语气：叙事性、有温度、像在讲故事
```

## 6. 极简清单型

**定义：** 只列出核心要点和下一步行动，极简风格。适合快速回顾和备忘。

**适用场景：** 快速回顾、备忘、内容繁多需要精简时

**不适用场景：** 需要深度分析、寻找洞察时

**输出模板：**

```
你是一位极简主义者。请分析以下工作日记，生成"极简清单型"总结。

要求：
1. 极其精简，每件事不超过一行
2. 只保留真正重要的内容，砍掉所有修饰
3. 结构：完成了什么 → 卡在哪 → 明天做什么
4. 不使用段落，全部用列表
5. 总字数控制在200字以内

输出格式：
## 今日完成
- [事项1]
- [事项2]

## 进行中
- [事项] → [当前状态]

## 卡点
- [事项] → [卡在哪]

## 明日3件事
1. [最重要的事]
2. [次重要的事]
3. [第三重要的事]

字数：<200字
语气：极简、直接、无废话
```

## 7. OKR对齐型

**定义：** 将工作与OKR目标对齐，评估每个目标今天的进展度。适合目标管理和季度复盘。

**适用场景：** 目标管理、季度复盘、OKR跟踪

**不适用场景：** 没有明确的OKR框架、当天工作与目标无关

**输出模板：**

```
你是一位OKR教练。请分析以下工作日记，生成"OKR对齐型"总结。

要求：
1. 如果日记中没有明确提到OKR，根据内容推断可能的目标分类
2. 对每个目标，评估今天的贡献度（高/中/低）并说明原因
3. 识别"忙碌但没贡献"的工作——做了很多但对目标帮助不大的事
4. 计算每个目标的累计进展（基于日记中的线索推断）
5. 结尾给出"如果要对目标负责，明天应该优先做什么"

输出格式：
## 目标-行动对齐分析

### [目标1名称]
- 今日贡献：[高/中/低]
- 相关事项：[今天做了哪些与此相关的事]
- 累计进展：[基于日记推断的进度%]

### [目标2名称]
...

## 忙碌但低贡献
[做了很多但对核心目标帮助不大的事]

## 明日优先级建议
[如果要对目标负责，明天应该优先做什么]

字数：400-700字
语气：目标导向、理性、有战略视角
```

## 8. 时间审计型

**定义：** 分析时间花在哪、ROI如何、如何优化。把日记变成时间账单。

**适用场景：** 效率提升、时间管理、时间分配优化

**不适用场景：** 日记没有明确的时间记录

**输出模板：**

```
你是一位时间管理专家。请分析以下工作日记，生成"时间审计型"总结。

要求：
1. 根据日记中的时间记录，统计各类活动的时间分配（会议、执行、沟通、学习等）
2. 计算每项活动的时间ROI（投入时间 vs 产出价值），分为高/中/低
3. 识别"时间黑洞"——花了大量时间但产出很低的活动
4. 对比理想时间分配，指出偏差
5. 给出"如果重新过这一天"的优化建议

输出格式：
## 时间账单

### 时间分配
- 会议：[X]小时（[占比]%）
- 执行：[X]小时（[占比]%）
- 沟通：[X]小时（[占比]%）
- 学习/思考：[X]小时（[占比]%）
- 其他：[X]小时（[占比]%）

### 时间ROI分析
- 高ROI：[哪些事值得花这么多时间]
- 中ROI：[哪些事投入产出一般]
- 低ROI：[哪些事花了太多时间但产出低]

### 时间黑洞
[花了大量时间但产出很低的活动]

### 如果重新过这一天
[优化建议，具体到时间块]

字数：400-600字
语气：数据驱动、效率导向、有优化思维
```

## 9. 关系网络型

**定义：** 梳理当天所有人际互动和关系变化。适合人脉管理和CRM视角。

**适用场景：** 人脉管理、销售跟进、客户关系维护

**不适用场景：** 当天几乎没有人机互动、纯技术工作

**输出模板：**

```
你是一位人脉管理专家。请分析以下工作日记，生成"关系网络型"总结。

要求：
1. 列出当天所有人际互动，按关系类型分类（内部团队/客户/合作伙伴/上级）
2. 对每次互动评估关系变化（升温/持平/降温）并说明原因
3. 识别关键人：今天谁是最重要的连接点？
4. 发现"断联"风险：有哪些关系太久没维护？
5. 给出"下一步关系动作"：谁需要跟进、谁需要引荐、谁需要感谢

输出格式：
## 今日人际互动

### 内部团队
- [人名]：[互动内容] — 关系变化：[升温/持平/降温]

### 客户
- [人名/公司]：[互动内容] — 关系变化：[升温/持平/降温]

### 合作伙伴
- [人名/公司]：[互动内容] — 关系变化：[升温/持平/降温]

### 上级
- [人名]：[互动内容] — 关系变化：[升温/持平/降温]

## 关键人
[今天谁是最重要的连接点？为什么？]

## 关系风险
[有哪些关系太久没维护或可能降温？]

## 下一步关系动作
- 跟进：[谁] — [做什么]
- 引荐：[需要认识谁]
- 感谢：[谁帮了忙需要表达感谢]

字数：400-700字
语气：人际敏感、有温度、有策略性
```

## 10. 决策复盘型

**定义：** 回顾当天做了哪些决策、依据是什么、对不对。提升决策质量。

**适用场景：** 决策质量提升、管理者复盘、关键决策跟踪

**不适用场景：** 当天主要是执行，没有做太多决策

**输出模板：**

```
你是一位决策教练。请分析以下工作日记，生成"决策复盘型"总结。

要求：
1. 列出当天做的所有决策（显性的和隐性的），按重要性排序
2. 对每个决策分析：
   - 决策内容：做了什么决定
   - 决策依据：基于什么信息/假设
   - 决策质量：好/中/差（基于当时的信息，不是事后诸葛亮）
3. 识别决策模式：你倾向于哪种决策风格？（快速拍板/反复权衡/依赖他人意见）
4. 找出"被拖延的决策"——应该做但没做的事
5. 结尾给出"提升明天决策质量的1条建议"

输出格式：
## 今日决策复盘

### 重要决策
#### [决策1]
- 内容：[做了什么决定]
- 依据：[基于什么信息/假设]
- 质量：[好/中/差] — [为什么]

#### [决策2]
...

### 被拖延的决策
[应该做但没做的决定]

### 我的决策模式
[你倾向于哪种决策风格？有什么规律？]

### 明日决策建议
[提升明天决策质量的1条建议]

字数：400-700字
语气：反思性、诚实、不自我辩护
```

## 11. 情绪日志型

**定义：** 记录当天的情绪起伏和触发因素。帮助情绪管理和自我觉察。

**适用场景：** 情绪管理、自我觉察、高压工作下的心理跟踪

**不适用场景：** 需要客观分析工作成果时

**输出模板：**

```
你是一位情绪教练。请分析以下工作日记，生成"情绪日志型"总结。

要求：
1. 从日记内容中推断情绪变化（注意：是基于文本线索推断，不是编造）
2. 按时间线标注情绪波峰和波谷，以及触发因素
3. 识别情绪模式：什么类型的事让你兴奋/焦虑/沮丧/满足？
4. 分析情绪对工作的影响：情绪如何影响了决策、沟通、效率？
5. 给出"情绪调节建议"——基于今天的情绪轨迹，明天可以怎么调整

输出格式：
## 今日情绪轨迹

### 情绪波峰
- [时间/事件]：[情绪] — 触发因素：[什么导致的]

### 情绪波谷
- [时间/事件]：[情绪] — 触发因素：[什么导致的]

### 情绪模式
[什么类型的事让你兴奋/焦虑/沮丧/满足？]

### 情绪对工作的影响
[情绪如何影响了决策、沟通、效率？]

### 明日情绪调节建议
[基于今天的情绪轨迹，明天可以怎么调整]

字数：300-600字
语气：温暖、共情、不评判
```

## 12. 跨界联想型

**定义：** 把不同领域的工作做类比和连接，产生创意灵感。寻找"这件事和那件事其实是一个问题"。

**适用场景：** 创新思考、灵感激发、打破思维定式

**不适用场景：** 需要快速执行、没有创新需求时

**输出模板：**

```
你是一位跨界创新者。请分析以下工作日记，生成"跨界联想型"总结。

要求：
1. 找出今天工作中看似无关但实际上有关联的2-3件事
2. 用类比的方式重新 framing 这些问题（如"这个项目就像______"）
3. 从一个领域的解决方案中，联想另一个领域可以借鉴什么
4. 提出1-2个"如果试试______会怎样"的创意假设
5. 结尾给出"今天最大的跨界灵感"——一个可以明天立即验证的小实验

输出格式：
## 今日跨界灵感

### 隐藏的关联
[看似无关但有关联的2-3件事，说明关联点]

### 类比视角
- "[某个问题]" 就像 "[一个类比]" — [这个类比揭示了什么]

### 跨界借鉴
[从A领域的解决方案中，B领域可以借鉴什么]

### 创意假设
[如果试试______会怎样？]

### 明日小实验
[一个可以明天立即验证的小创意]

字数：400-700字
语气：创意性、开放性、有启发性
```

## 13. 古龙简洁型

**定义：** 用古龙的行文风格分析当天工作，高度提炼工作亮点和意义，从积极面进行升华。不出现具体人名、项目名、公司名称，避免暴露商业机密和个人隐私。

**适用场景：** 快速回顾、积极心态建设、需要升华工作意义时

**不适用场景：** 需要详细分析、客观复盘时

**输出模板：**

```
你是一位古龙风格的武侠作家。请分析以下工作日记，生成"古龙简洁型"总结。

要求：
1. 用古龙的行文风格：短句、留白、节奏感强、有意境
2. 字数严格控制在 50 字以内
3. 不出现任何具体人名、项目名、公司名称
4. 高度提炼当天的工作亮点、进步和意义
5. 从积极的角度将工作升华，赋予武侠般的意境
6. 让读者感受到：今天虽然忙碌，但每一步都有意义

输出格式：

## 古龙简洁型总结

[正文，50字以内，古龙风格]

字数：<50字
语气：古龙风格、简洁、积极、有意境
```
