# Work Summary Skill

从日记、会议记录、碎片笔记等素材中，生成多风格的深度工作总结。核心理念：**不要复述做了什么，要说出发现了什么。**

## 交互流程

### Step 1 — 获取素材

**1.1 确认时间范围（必须询问）**

无论用户是否提到了日期，都必须先确认时间范围：

```
你想总结哪段时间的工作？

请输入日期或时间范围，例如：
- 2026-05-12（某一天）
- 本周（周一到周日）
- 2026-05（整月）
- 2026-05-10 到 2026-05-12（自定义范围）
```

如果用户在启动时已指定了日期，也要向用户确认：

```
你提到了 2026-05-12，请确认是要总结这一天，还是其他时间范围？
```

**1.2 确定日记文件夹**

如果不知道用户的日记文件夹路径，询问用户：

```
你的日记文件放在哪个文件夹？请提供完整路径。
```

获取后记住该路径，下次直接使用。

**1.3 自动查找日记文件**

根据时间范围和日记文件夹路径，自动查找匹配的文件。

查找规则：
- 文件名包含"日记"且包含日期（如 `日记2026-05-12.md`）
- 按日期匹配用户指定的时间范围
- 对于日期范围（如"本周"），查找该范围内所有日期的日记文件

**1.4 处理查找结果**

- **找到文件**：告知用户找到了哪些文件，继续 Step 1.5
- **未找到文件**：告知用户未找到日记文件，请用户确认文件夹路径是否正确，或提供具体文件路径
- **找到部分文件**：告知用户找到了哪些、缺少哪些日期，询问是否继续

**1.5 读取素材**

读取找到的所有日记文件内容。读取后，快速分析内容特征（会议数量、涉及人物、项目类型、情绪基调等），为 Step 2 的风格推荐做准备。

**支持的其他素材方式（如果自动查找不可用）：**

- **直接粘贴内容** — 用户直接粘贴日记/笔记内容
- **指定具体文件** — 用户提供具体的文件路径

### Step 2 — 选择风格

始终向用户展示全部 13 种风格，同时根据素材内容智能推荐 1-3 种，并简短说明推荐理由。

**推荐逻辑：**
- 会议多、项目推进为主 → 推荐「成果导向型」+「风险预警型」
- 问题多、卡点多 → 推荐「风险预警型」+「模式发现型」
- 学习、讨论、思考为主 → 推荐「学习成长型」+「跨界联想型」
- 内容杂、信息量大 → 推荐「模式发现型」+「极简清单型」
- 人际互动多 → 推荐「关系网络型」
- 决策多 → 推荐「决策复盘型」
- 情绪波动明显 → 推荐「情绪日志型」

**展示格式：**

```
推荐风格：X（理由）、X（理由）

13种总结风格：

1. 模式发现型   — 找规律、重复问题、趋势变化
2. 成果导向型   — 聚焦完成的事，结构清晰适合汇报
3. 风险预警型   — 识别潜在问题、依赖风险、卡点
4. 学习成长型   — 提炼认知升级、经验教训、方法论
5. 叙事故事型   — 把流水账串成有起承转合的故事
6. 极简清单型   — 只列核心要点和下一步行动
7. OKR对齐型   — 将工作与目标对齐，评估进展度
8. 时间审计型   — 分析时间花在哪、ROI如何
9. 关系网络型   — 梳理人际互动和关系变化
10. 决策复盘型  — 回顾决策、依据、对错
11. 情绪日志型  — 记录情绪起伏和触发因素
12. 跨界联想型  — 跨领域类比，产生创意灵感
13. 古龙简洁型  — 古龙笔法，50字内升华工作意义

请选择风格编号（可多选，如 1,3,13）：
```

支持用户选择多种风格，一次生成多个视角的总结。

### Step 3 — 生成总结

1. 加载 `references/styles.md` 中对应风格的完整定义和 prompt 模板
2. 读取素材内容
3. 如果用户选择了多个日期的素材，先做跨天对比分析
4. 按选定风格的 prompt 模板生成总结
5. 展示给用户

### Step 4 — 输出方式（必须执行，不得跳过）

**重要：Step 3 生成总结后，必须立即执行 Step 4，不得跳过或省略。**

总结展示后，立即询问输出方式：

```
总结已生成。请选择输出方式：

1. 仅在对话中展示（默认）
2. 追加到日记文件末尾
3. 生成独立文件

输入编号：
```

- **方式1**：直接在对话中展示，不修改任何文件
- **方式2**：追加到原始日记文件的 `# 总结` 部分下方
- **方式3**：生成独立文件，命名规则为 `总结YYYY-MM-DD.md`，放在日记文件夹中

如果选择方式3，询问文件保存路径（默认与日记同目录）。

## 核心原则

1. **观点优先**：每篇总结必须有一个核心观点/发现，不能只是流水账的缩写版
2. **证据支撑**：观点必须有日记中的具体事件作为证据
3. **反直觉**：优先找"和预期相反"的点，这才是最有价值的洞察
4. **对比视角**：如果有多个日期的素材，一定要做跨天对比，找趋势
5. **一句话提炼**：每种风格的总结开头都有一句核心提炼
6. **行动导向**：总结末尾必须有可执行的行动启示，不是空泛的感想

## 注意事项

- 如果日记内容太少（少于3条记录），告知用户素材不足，建议补充
- 如果用户没有指定日记文件路径，主动询问
- 生成总结前不要修改原始日记文件
- 输出到文件前必须征得用户确认

## Obsidian CLI 集成

当用户的 vault 启用了 Obsidian CLI（命令 `obsidian` 可用）时，优先使用 CLI 命令操作 vault 文件：

**读取文件：**
```
obsidian read path="11日记文件夹/日记2026-05-12.md"
```

**追加内容到文件：**
```
obsidian append path="11日记文件夹/日记2026-05-12.md" content="总结内容"
```

**创建文件：**
```
obsidian create path="11日记文件夹/总结2026-05-12.md" content="总结内容"
```

**搜索文件：**
```
obsidian search query="日记2026-05"
```

**列出文件：**
```
obsidian files folder="11日记文件夹"
```

**重要：**
- Obsidian CLI 要求 Obsidian app 正在运行
- 路径是相对于 vault 根目录的路径，不需要完整文件系统路径
- 如果 CLI 不可用（命令不存在或 Obsidian 未运行），回退到直接文件操作
- 首次使用时可运行 `obsidian help` 检测 CLI 是否可用
