# MD 待办提取器

从 Markdown 文件中智能提取待办事项，并添加到 macOS 系统应用中。

## 支持的文件格式

- `.md` 文件（单个文件或文件夹）

## 分类规则

根据以下规则将待办事项分类到不同的系统应用：

| 系统应用 | 判断标准 |
|---------|---------|
| **日历 (Calendar)** | 在同一个半天内（≤4小时）且有明确的开始时间点和结束时间点的会议/讨论/汇报等活动 |
| **提醒事项 (Reminders)** | 时间跨度超过半天，或只有截止时间而无明确开始时间 |
| **备忘录 (Notes)** | 没有明确时间，但很重要 |
| **跳过** | 既没有明确时间，也不重要 |

### 时间判断优先级

1. 同时有明确开始和截止时间，且跨度≤4小时 → **日历**
2. 有截止时间但无明确开始时间，或跨度>4小时 → **提醒事项**
3. 无明确时间但很重要 → **备忘录**
4. 既没时间也不重要 → **跳过**

### 日历事项标题命名规则（重要）

**日历事项代表的是"讨论"、"会议"、"汇报"等活动**，而不是执行的任务。

**错误示例：**
- 原文："建立周报机制（每周二下午收取文旅客户周报汇报）"
- 错误标题："建立周报机制" ← 这是任务名称，不是日历事件

**正确示例：**
- 原文："建立周报机制（每周二下午收取文旅客户周报汇报）"
- 正确标题："文旅客户周报汇报" ← 这是讨论/汇报的主题
- 时间：每周二 14:00-15:00（ recurring）
- 描述可包含：需要汇报的三项核心内容

**判断技巧：**
- 如果待办内容涉及"讨论"、"会议"、"汇报"、"评审"、"汇报"等 → 很可能是日历事项
- 日历标题应该是"关于XXX的讨论/汇报"，而不是"做XXX"

## 工作流程

### 步骤 1：接收用户输入

用户需提供：
- md 文件路径，或包含 md 文件的文件夹路径
- 可选：指定分析的时间范围（自然语言描述，如"上周"、"5月1日到5月7日"）

**提示用户输入：**
```
请提供 md 文件路径或包含 md 文件的文件夹路径。
如需限定时间范围，请说明（如"分析上周创建的文件"）。
```

### 步骤 2：扫描文件

1. 如果是**文件路径**：
   - 检查是否为 `.md` 文件
   - 读取文件内容

2. 如果是**文件夹路径**：
   - 扫描文件夹及子文件夹中的所有 `.md` 文件
   - 如果用户指定了时间范围，在该范围内筛选文件
   - 如果未指定，分析所有 `.md` 文件

**时间范围筛选逻辑：**
- 用户使用系统本地时区
- 按文件创建时间（`stat` 命令获取 `birthtime`）在指定范围内筛选

### 步骤 3：AI 提取待办事项

分析每个 md 文件内容，识别潜在的待办事项。

**识别原则（无固定标记，AI 自行判断）：**
- 包含动作项的句子（如"需要做某事"、"记得去..."）
- 带有意图动词的陈述（"要"、"应该"、"记得"）
- 明确的任务描述（即使没有 TODO 标记）
- 用户明确标注为待办的内容

**每条待办记录格式：**
```json
{
  "content": "待办内容原文",
  "source_file": "文件名.md",
  "importance": "高/中/低",
  "has_start_time": true/false,
  "has_end_time": true/false,
  "start_time": "ISO 时间或 null",
  "end_time": "ISO 时间或 null",
  "classification": "calendar/reminders/notes/skip",
  "reason": "分类原因说明"
}
```

### 步骤 4：显示分类结果，待用户确认

**必须使用以下精确格式输出分类结果表格：**

```
📋 提取到的待办事项：

| # | 内容摘要 | 分类 | 原因 |
|---|---------|------|------|
| 1 | 完成项目报告 | 📅 日历 | 同半天内有明确开始和结束时间 |
| 2 | 提交财务报表 | ⏰ 提醒事项 | 有截止时间，无开始时间 |
| 3 | 学习 React 基础 | 📝 备忘录 | 重要但无明确时间 |
| 4 | 整理桌面 | ⏭️ 跳过 | 不重要且无明确时间 |

分类汇总：📅 日历 X 项 | ⏰ 提醒事项 X 项 | 📝 备忘录 X 项 | ⏭️ 跳过 X 项

请确认哪些需要添加到系统（输入编号，多选用逗号分隔）：
- 输入 "1,2,3" 确认第 1、2、3 项
- 输入 "all" 确认全部
- 输入 "none" 取消全部
```

**重要：分类列必须使用以下图标之一：**
- 📅 日历 - 表示要添加到日历
- ⏰ 提醒事项 - 表示要添加到提醒事项
- 📝 备忘录 - 表示要添加到备忘录
- ⏭️ 跳过 - 表示不重要且无时间，跳过不添加

**分类判断规则（按优先级）：**
1. 同时有明确开始和截止时间，且跨度≤4小时 → 📅 日历
2. 有截止时间但无明确开始时间，或跨度>4小时 → ⏰ 提醒事项
3. 无明确时间但很重要 → 📝 备忘录
4. 既没时间也不重要 → ⏭️ 跳过

### 步骤 5：获取用户选择的日历/列表

根据待办分类，显示相应的可用列表供用户选择：

**日历 (Calendar)：**
- 调用 AppleScript 或 `calendarcli` 获取用户日历列表
- 显示日历名称，让用户选择目标日历

**提醒事项 (Reminders)：**
- 调用 AppleScript 获取用户的提醒事项列表
- 显示列表名称，让用户选择目标列表

**备忘录 (Notes)：**
- 调用 AppleScript 获取用户的备忘录账户和文件夹
- 显示笔记名称，让用户选择目标位置

**提示用户：**
```
选择目标位置：
- 日历待办 1, 2, 3，请选择目标日历：
  1. 工作
  2. 个人
  3. 家庭
  > 2

- 提醒事项请选择目标列表：
  1. 日常任务
  2. 工作待办
  > 1
```

### 步骤 6：添加到系统

根据用户选择，使用 AppleScript 将待办事项添加到相应的系统应用。

**重要：执行前先获取用户的日历/提醒事项列表供选择**

**获取日历列表：**
```applescript
tell application "Calendar"
  set calendarNames to title of every calendar
end tell
```

**获取提醒事项列表：**
```applescript
tell application "Reminders"
  set listNames to name of every list
end tell
```

**获取备忘录文件夹：**
```applescript
tell application "Notes"
  set folderNames to name of every folder
end tell
```

**添加到日历：**
```applescript
tell application "Calendar"
  tell calendar "目标日历名称"
    make new event at end with properties {summary:"待办标题", description:"待办描述", start date:date "2026/5/10 9:00:00", end date:date "2026/5/10 12:00:00"}
  end tell
end tell
```

**添加到提醒事项：**
```applescript
tell application "Reminders"
  tell list "目标列表名称"
    make new reminder at end with properties {name:"待办标题", body:"待办描述", due date:date "2026/5/15 18:00:00"}
  end tell
end tell
```

**添加到备忘录：**
```applescript
tell application "Notes"
  tell folder "目标文件夹"
    make new note at end with properties {name:"待办标题", body:"待办描述"}
  end tell
end tell
```

**删除测试数据（执行删除前必须用户确认）：**
```applescript
-- 删除日历事件
tell application "Calendar"
  delete (every event of calendar "目标日历名称" whose summary contains "测试标记")
end tell

-- 删除提醒事项
tell application "Reminders"
  delete (every reminder of list "目标列表名称" whose name contains "测试标记")
end tell

-- 删除备忘录
tell application "Notes"
  delete (every note of folder "目标文件夹" whose name contains "测试标记")
end tell
```

**建议：为测试数据添加 [测试] 前缀，便于后续识别和删除**

### 步骤 7：完成确认

显示添加结果摘要：

```
✅ 已成功添加 3 项到系统：

📅 日历 - "工作"：
  - 完成项目报告 (5月10日 9:00-12:00)

⏰ 提醒事项 - "工作待办"：
  - 提交财务报表 (截止: 5月15日 18:00)

📝 备忘录 - "工作笔记"：
  - 学习 React 基础

⚠️ 跳过 1 项（不重要且无明确时间）
```

## 错误处理

| 错误情况 | 处理方式 |
|---------|---------|
| 文件不存在 | 提示用户检查路径，提供示例路径格式 |
| 不是 md 文件 | 提示用户仅支持 .md 格式 |
| 无法访问日历/提醒事项 | 提示用户检查系统权限设置 |
| 添加失败 | 显示具体错误信息，提示用户手动添加 |

## 权限要求

首次使用需要授予以下 macOS 权限：
- **日历**：系统偏好设置 → 隐私与安全性 → 日历
- **提醒事项**：系统偏好设置 → 隐私与安全性 → 提醒事项
- **备忘录**：系统偏好设置 → 隐私与安全性 → 备忘录
